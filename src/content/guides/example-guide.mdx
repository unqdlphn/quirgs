This is a scaffold guide. It exists to exercise the net-new `guides` content
collection end-to-end: schema validation, `getStaticPaths`, MDX rendering through
`BaseLayout`, the strict `/*` CSP, and the clean `/guides/example-guide/` URL.
Replace or delete it once a real guide ships via Keystatic.

## What this proves

- The `guides` collection in `src/content.config.ts` validates frontmatter.
- `src/pages/guides/[slug].astro` renders the MDX body in the terminal UI.
- The page is reachable at a directory-style URL, distinct from the legacy
  `public/guides/*.html` archive.

## Markdown sanity check

Inline `code`, a fenced block, and a list — to confirm the terminal-body
typography in `BaseLayout` styles MDX as expected:

```bash
npm run build   # emits /guides/example-guide/index.html
```

1. Authored as MDX from the start — nothing to convert.
2. Keystatic-managed (edit on a branch, never `main`).
3. Drops cleanly into the existing static build.
