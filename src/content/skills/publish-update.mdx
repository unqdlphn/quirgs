## What it does

Publish Update audits a music catalog's metadata against ISRC, UPC, DDEX, and distribution requirements. It identifies every gap and error across required fields — ISRCs (missing, malformed, or duplicated), UPCs, title formatting, artist credits, genre classification, ISWC for compositions, and rights ownership data. Each error is classified by type and field priority, with a field-level quality score that shows where the catalog stands overall.

After the audit, the skill enriches incomplete records by inferring or sourcing correct values from authoritative references: ISRC lookup databases, ISWC registry patterns, and standard genre taxonomies. The output is an audit report paired with an import-ready corrected data set, formatted for direct re-upload to a distributor.

## Frameworks covered

- ISRC standard (ISO 3901) — sound recording identifiers
- ISWC standard (T-000.000.000-C) — composition identifiers
- UPC/EAN-13 — release identifiers
- DDEX Data Dictionary — field naming and format conventions
- MusicBrainz metadata schema — canonical field structure

## When to use it

Run Publish Update before any distribution event — new release, catalog re-delivery, sync pitch submission, or catalog sale. Clean metadata is a prerequisite for all downstream operations: DDEX registration (publish-broadcast), sync licensing (publish-license), and royalty reconciliation (publish-income). Also run it when a new catalog is acquired, since inherited metadata is rarely clean.
