## What it does

Publish Income compares incoming royalty statements — from DSPs, distributors, and performing rights organizations — against an internal catalog ledger and flags every discrepancy it finds. Each flag is classified on a D1-D6 severity scale: D1 for minor rounding variances, up to D6 for structural conflicts that require legal or accounting review. The skill outputs a structured discrepancy report with flagged line items, recommended actions per flag, a reconciliation summary, and a master ledger template pre-filled from the statement data.

When no ledger exists yet, the skill builds one from the statement, establishing a baseline for all future reconciliation cycles.

## Frameworks covered

- PRO distribution reporting (ASCAP, BMI, SESAC, PRS, SOCAN)
- DSP royalty statement formats (Spotify, Apple Music, YouTube, Amazon, TikTok)
- Distributor summary formats (DistroKid, TuneCore, CD Baby, AWAL, Amuse)
- Statement-to-Ledger reconciliation protocol

## When to use it

Run Publish Income whenever a new royalty statement arrives. It's the entry point for the quarterly catalog review cycle — publish-harvest consumes the reconciled ledger as its primary input. Also use it when a payment looks wrong, streams and revenue are misaligned, or multiple source statements need consolidating into a single view. If a track is missing from a statement entirely, the skill will surface that as a D4 flag for investigation.
