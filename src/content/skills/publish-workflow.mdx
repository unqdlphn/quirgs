## What it does

Publish Workflow is the master entry point for the PUBLISH Bundle. It implements the PUBLISH Loop — a seven-pillar weekly operating cadence for music publishers — assessing what's due in the current period, routing to the appropriate domain skill, and enforcing HITL checkpoints between every stage.

On each run, the workflow asks what data and tasks are available: new royalty statements, catalog updates, registrations pending, sync briefs, new releases, provenance logs, or a quarterly review cycle. It then builds a prioritized task queue, routes each item to the correct domain skill, and tracks completion. Partial data is handled by design — the workflow documents what's missing, proceeds with what's available, and flags deferred items for the next session.

## The PUBLISH Loop

Each pillar maps to a domain skill and runs on its natural cadence:

| Pillar | Skill | Cadence |
|---|---|---|
| Income | publish-income | Monthly / per statement |
| Update | publish-update | Per release / per distribution event |
| Broadcast | publish-broadcast | Per new release |
| License | publish-license | Per sync brief |
| Provenance | publish-provenance | Per release |
| Shield | publish-shield | Per release / policy review |
| Harvest | publish-harvest | Quarterly |

## Frameworks covered

- PUBLISH Loop — seven-pillar music publishing operating system
- HITL checkpoint protocol — human review required between every stage
- Music publishing SOPs — standard operating procedures for catalog management
- Weekly cadence management — triage, routing, and deferral handling

## When to use it

Start every publishing session with Publish Workflow. It's the single entry point that prevents the common failure mode of jumping straight into one task while ignoring equally urgent work elsewhere in the operation. Even when only one pillar is active (a sync brief arrives mid-week), routing through the workflow surfaces any dependencies — a brief cannot be pitched if Shield hasn't cleared the candidate tracks, which requires Provenance to have run first.

For owners who want a weekly executive summary rather than task-by-task output, the workflow produces a one-page operating report at the end of each session.
