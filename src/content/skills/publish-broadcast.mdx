## What it does

Publish Broadcast extracts and validates all data required to register a release with DDEX delivery networks and with performing rights organizations. It maps every required field for both registration types, surfaces gaps (missing IPI numbers, unregistered co-writers, split conflicts between PROs), and flags anything that would cause a rejection before submission.

For DDEX, it produces a validated ERN 4.1 data packet covering sound recording identifiers, deal terms, and release metadata. For PROs, it generates separate composition registration records for each rights society involved, handling split-PRO scenarios where co-writers are members of different organizations. All outputs are submission-ready — formatted for direct upload or entry into PRO web portals.

## Frameworks covered

- DDEX ERN 4.1 — Electronic Release Notification standard for DSP delivery
- ASCAP Works Registration — composition and recording registration
- BMI Songview — works registration and share reporting
- SESAC, PRS for Music, SOCAN — international PRO registration protocols
- ISRC (ISO 3901) — sound recording identifiers required for DDEX

## When to use it

Run Publish Broadcast after publish-update confirms metadata is clean — DDEX and PRO registration will fail on bad ISRCs or missing credits. Use it for every new release before it goes to distributors, and whenever a composition is being registered for the first time. It's particularly important for co-written tracks: split-PRO scenarios (ASCAP composer + BMI co-writer) require separate registration packets for each society, and the skill handles that routing automatically.
